# Local HTML Annotations

## Overview

Use this skill to inject a self-contained annotation layer into a static HTML report. The injected UI works on local `file://` pages without a server: users can select text, add comments, highlight the original text, manage comments in a side panel, and export/import annotations as JSON.

## Quick Start

Run the bundled injector:

```bash
python3 /Users/luoji/.codex/skills/local-html-annotations/scripts/inject_local_annotations.py /absolute/path/to/report.html
```

The script is idempotent. It removes any previous annotation blocks marked by `LOCAL_ANNOTATIONS_CSS_START/END` and `LOCAL_ANNOTATIONS_JS_START/END`, then reinjects the latest CSS and JS.

## Workflow

1. Identify the target static HTML file.
2. If the HTML is generated from Markdown or another renderer, make source-content edits first and render the HTML before injecting annotations.
3. Run `scripts/inject_local_annotations.py target.html`.
4. Validate the result:
   - Check the HTML contains `LOCAL_ANNOTATIONS_CSS_START` and `LOCAL_ANNOTATIONS_JS_START`.
   - Extract the injected script and run `node --check` when Node is available.
   - Optionally open the HTML and confirm the bottom-right `批注` button appears.
5. Tell the user how to use it and where annotations are stored.

## User Instructions To Provide

Give the user concise usage instructions:

- Click the bottom-right `批注` button to open the panel.
- Select text in the report body, then click the floating `添加批注` button.
- Enter the note and save.
- Use the side panel to locate, edit, delete, export, or import annotations.
- Export JSON for backup or sharing.

Warn the user:

- Annotations are stored in browser `localStorage` using the current file path as part of the key.
- Clearing browser data, changing browsers, or changing the file path can hide local annotations.
- Re-rendering the HTML may remove the injected annotation UI; run the injector again after each render.
- If the underlying text changes substantially, old exported annotations may not perfectly re-anchor, but the selected text and note remain visible in the panel.

## Data Model

The injected script stores annotations in `localStorage` under:

```text
sngamma.localAnnotations:<location.pathname>
```

Each annotation includes:

- selected text
- note text
- section title/id
- CSS-like block path within `.doc-body`
- occurrence index for repeated selected text
- timestamps
- source URL/title

The export button writes a JSON payload with schema `sngamma-local-annotations-v1`.

## Requirements And Limits

- The HTML should contain an article/body container with class `.doc-body`. This matches reports generated by the `notion-html-report` skill.
- The tool is serverless and browser-local; it does not write annotation content back into the Markdown source.
- For non-`.doc-body` HTML, patch the script selector before injection or add a `.doc-body` wrapper around the content.
- The UI is designed for reading/review workflows, not collaborative real-time editing.

## Validation Commands

After injection, a robust local check is:

```bash
python3 - <<'PY'
from pathlib import Path
html = Path("report.html").read_text()
assert "LOCAL_ANNOTATIONS_CSS_START" in html
assert "LOCAL_ANNOTATIONS_JS_START" in html
start = html.index("<!-- LOCAL_ANNOTATIONS_JS_START -->")
end = html.index("<!-- LOCAL_ANNOTATIONS_JS_END -->")
block = html[start:end]
Path("/tmp/local_html_annotations_check.js").write_text(
    block[block.index("<script>") + len("<script>"):block.rindex("</script>")]
)
PY
node --check /tmp/local_html_annotations_check.js
```

If `node` is unavailable, at minimum confirm the marker comments and the `批注` button text exist.
